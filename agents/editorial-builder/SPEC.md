# Editorial Builder — Output Specification

## File Format

Every article is an `.astro` file that imports `OlspLayout` and Gold Master components. The layout, CSS tokens, TOC, and JavaScript live in `OlspLayout`, not in the article file.

```astro
---
export const prerender = true;

import OlspLayout from "../../components/olsp-standard/OlspLayout.astro";
import Callout from "../../components/olsp-standard/Callout.astro";
import GoldMasterQuote from "../../components/olsp-standard/GoldMasterQuote.astro";
import FaqItem from "../../components/olsp-standard/FaqItem.astro";
import AuthorBox from "../../components/olsp-standard/AuthorBox.astro";
import SiteFooter from "../../components/olsp-standard/SiteFooter.astro";

const pageTitle = "...";
const pageDescription = "...";

const tocLinks = [
  { href: "#intro", label: "Introduction" },
  // ... section links
];
---

<OlspLayout title={pageTitle} description={pageDescription} canonical="https://olsp.profitandprivilege.com/{type}/{slug}/" tocLinks={tocLinks}>

  <section id="intro">
    <!-- Hero Tag, h1, VerdictBox, Methodology -->
  </section>

  <!-- Content sections -->

  <section id="faq">
    <h2>Frequently Asked Questions</h2>
    <FaqItem question="..." answer="..." />
  </section>

  <section id="author">
    <h2>About the Author</h2>
    <AuthorBox />
  </section>

  <section id="sources">
    <h2>Sources &amp; References</h2>
    <ul class="pill-list">
      <!-- source links -->
    </ul>
    <SiteFooter />
  </section>

</OlspLayout>
```

## CSS Block

The CSS block lives in `OlspLayout.astro` — articles do NOT include their own CSS. OlspLayout provides all CSS tokens, typography, layout grid, and component styles.

## JavaScript Block

The JavaScript block lives in `OlspLayout.astro` — articles do NOT include their own JS. OlspLayout provides mobile TOC toggle, scroll-spy, TOC link-close, and quiz evaluation logic.

## Head Section Per Type

The `<head>` block is generated by `OlspLayout` from the article's props. Articles do NOT include their own `<head>`.

### Review
- No Open Graph tags, no Twitter Card tags, no JSON-LD (by design)

### Blog
- Must include Open Graph tags (`og:title`, `og:description`, `og:url`, `og:type`, `og:site_name`)
- Must include Twitter Card tags
- Must include JSON-LD (`Article` + `FAQPage`)

### Roundup
- No Open Graph tags, no Twitter Card tags, no JSON-LD (same as reviews)

## Section Order Per Type

### Blog Article
1. `#intro` — HeroTag, h1, VerdictBox, h3
2. QuoteBanner #1
3. Standard CTA #1
4. Numbered body sections (3–7)
5. QuoteBanner #2 (mid-article)
6. More body sections
7. QuoteBanner #3 (pre-FAQ)
8. `#faq` — FaqItem components
9. Standard CTA #2
10. `#author` — AuthorBox
11. `#sources` — PillList, disclaimer, SiteFooter

### Review
1. `#intro` — HeroTag, h1, VerdictBox, Methodology
2. CTA Card #1
3. Body sections (overview, design, performance, ux, comparison, proscons, history, recommend, buy, verdict)
4. CTA Card #2
5. `#faq`
6. CTA Card #3
7. `#author`
8. `#sources`

### Roundup
1. `#intro` — HeroTag, h1, VerdictBox
2. CTA Card #1
3. Body sections
4. CTA Card #2
5. `#faq`
6. CTA Card #3
7. `#author`
8. `#sources`
