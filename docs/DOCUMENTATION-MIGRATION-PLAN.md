## 2. Classification Rules

### Production Documentation

Documents that are actively referenced by code, agents, or the pipeline. These are the source of truth for the system.

**Criteria:**
- Referenced by `agents/*/PROMPT.md`, `pipeline/PROMPT.md`, or `src/`
- Contains current specifications, contracts, or architecture
- Required for the system to function

### Reference Documentation

Documents that provide context, background, or supplementary information. Not directly referenced by code but useful for understanding the system.

**Criteria:**
- Contains research briefs, data assets, or asset libraries
- Provides context for decision-making
- Used by humans, not machines

### Historical Documentation

Documents that record past states, decisions, or analyses. Useful for understanding how the system evolved but not required for current operation.

**Criteria:**
- Documents a past state or decision
- Superseded by current documentation
- Useful for understanding history

### Archive

Documents that are no longer actively used but may be needed for reference or compliance. Preserved but not maintained.

**Criteria:**
- Never referenced by current code
- Superseded by newer documentation
- Contains implementation details that are no longer accurate

### Safe to Delete

Documents that have no value, are duplicates, or were never completed.

**Criteria:**
- Experimental or POC documents that were never implemented
- Point-in-time snapshots superseded by current state
- Incomplete or abandoned documents
- Duplicates of existing documents
